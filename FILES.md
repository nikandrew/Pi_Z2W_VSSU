ИСПОЛЬЗОВАННЫЕ ТЕХНОЛОГИИ:

Python:
  • asyncio - асинхронное выполнение
  • subprocess - запуск системных команд
  • logging - логирование
  • pathlib - работа с файлами
  • serial - работа с RS-485
  • gpiozero - управление GPIO

Системные утилиты:
  • rpicam-vid - запись видео (rpicam-apps)
  • ffmpeg - разбиение видео
  • bash - скрипты установки

ОС: Raspberry Pi OS (64-bit)
Python: 3.8+

БЫСТРАЯ СПРАВКА КОМАНД:

На Raspberry Pi:
  # Установка
  bash install.sh
  
  # Проверка
  python3 checklist.py
  
  # Тестирование
  python3 test_components.py
  sudo python3 test_rs485.py
  
  # Запуск
  python3 main.py
  
  # С логами
  python3 main.py 2>&1 | tee run.log

На компьютере (с USB RS-485):
  # Отправка команды
  python3 host_simulator.py --port /dev/ttyUSB0
  
  # Циклический тест
  python3 host_simulator.py --port /dev/ttyUSB0 --loop

═══════════════════════════════════════════════════════════

СТРУКТУРА RS-485 ОБЩЕНИЯ:

Хост → Pi:
  b"\x00\x01" → запуск записи видео

Pi → Хост:
  b"recording_complete" → запись завершена и сохранена

═══════════════════════════════════════════════════════════

УРОВНИ ЛОГИРОВАНИЯ:

INFO    - нормальная работа (записано видео, разбиено и т.д.)
DEBUG   - подробная отладочная информация
WARNING - предупреждения (опциональный компонент не найден)
ERROR   - ошибки (не удалось записать видео, GPIO ошибка)

Логи сохраняются в: logs/camera_system.log
