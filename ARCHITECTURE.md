## 📊 Состояния системы

```
┌────────────────────────────────────────────────────────────┐
│                     IDLE (ожидание)                        │
│                                                            │
│  • RS485Handler слушает
│  • GPIO17 = LOW (приём)
│  • CPU: ~5-10%
│  • Ожидание команды 'start'
└────────────────────────────────────────────────────────────┘
                           ▲
                           │ 'start' получена
                           ▼
┌────────────────────────────────────────────────────────────┐
│                 RECORDING (запись видео)                  │
│                                                            │
│  • rpicam-vid запущен в subprocess
│  • GPIO17 = LOW (приём от камеры)
│  • CPU: ~50-60% (запись + обработка)
│  • RAM: +20MB (буфер видео)
│  • Длительность: ~60 сек
└────────────────────────────────────────────────────────────┘
                           │
                           ▼
┌────────────────────────────────────────────────────────────┐
│              SPLITTING (разбиение видео)                  │
│                                                            │
│  • ffmpeg запущен (или простое разбиение)
│  • GPIO17 = LOW
│  • CPU: ~30-40%
│  • Длительность: ~20-40 сек (зависит от размера)
└────────────────────────────────────────────────────────────┘
                           │
                           ▼
┌────────────────────────────────────────────────────────────┐
│              SENDING (отправка результата)                │
│                                                            │
│  • GPIO17 = HIGH (передача)
│  • Отправка: b'recording_complete'
│  • Переключение обратно: GPIO17 = LOW
│  • Длительность: ~10 мс
└────────────────────────────────────────────────────────────┘
                           │
                           ▼
┌────────────────────────────────────────────────────────────┐
│                     IDLE (ожидание)                        │
└────────────────────────────────────────────────────────────┘
```

## 🔄 Поток выполнения команды 'start'

```
Временная шкала:
t=0 сек      - Команда получена
│
├─ t=0.1 сек   - Запуск rpicam-vid
│    ├─ VIDEO_DURATION_MS = 60_000 мс (1 минута)
│    │
├─ t=60 сек    - Видео завершено, сохранено
│    │
│    ├─ Проверка размера файла
│    │    ├─ Если ≤ 50 МБ: разбиение не требуется
│    │    └─ Если > 50 МБ: запуск ffmpeg
│    │
├─ t=90 сек    - Разбиение завершено (если требуется)
│    │
├─ t=90.05 сек - GPIO17 → HIGH (режим передачи)
│    │
├─ t=90.07 сек - Отправка b'recording_complete'
│    │
├─ t=90.08 сек - GPIO17 → LOW (режим приема)
│    │
└─ t=90.1 сек  - Команда завершена

Результат:
✓ videos/video_YYYYMMDD_HHMMSS.h264  [15-20 МБ]
✓ videos/video_YYYYMMDD_HHMMSS_part_0.h264  [50 МБ] (если нужно)
✓ videos/video_YYYYMMDD_HHMMSS_part_1.h264  [50 МБ] (если нужно)
✓ Ответ отправлен по RS-485: b'recording_complete'
```

## 📈 Производительность

```
Ресурсы на Raspberry Pi Zero 2W:

IDLE (ожидание):
  CPU: 3-5%
  RAM: ~30 МБ
  Сетевая нагрузка: ~100 байт/сек

RECORDING (запись 1 мин):
  CPU: 45-60%
  RAM: ~50 МБ (+20 МБ буфер)
  Размер видео: ~15-20 МБ
  Время: ~60 сек (в реальном времени)

SPLITTING (разбиение):
  CPU: 30-45%
  RAM: ~35 МБ
  Время: ~20-40 сек (зависит от ffmpeg)
  Скорость: ~0.5 GB/s

Общая последовательность:
  Idle → Record (60s) → Split (30s) → Send (0.1s) → Idle
  = Минимум 90 сек между командами
```

## 🔒 Потокобезопасность

```
Shared Resources (защита):
├─ serial.Serial (RS485Handler)
│  └─ Доступ только из asyncio event loop ✓
├─ GPIO17
│  └─ Управление через gpiozero (потокобезопасно) ✓
├─ Файловая система (видео)
│  └─ Уникальные имена с timestamp (конфликты исключены) ✓
└─ subprocess (rpicam-vid, ffmpeg)
   └─ Запуск через asyncio.create_subprocess_exec ✓
```

## 🛡️ Обработка ошибок

```
Уровень 1: RS-485 ошибка
└─ Логируется, буфер очищается, ждем новую команду

Уровень 2: Открытие UART порта
└─ Система отказывает в запуске (критическая ошибка)

Уровень 3: Запись видео
├─ rpicam-vid не найден → Логируется, система ждет
├─ Камера заблокирована → timeout, повтор
└─ Ошибка записи → Логируется, отправка ERROR ответа

Уровень 4: Разбиение видео
├─ ffmpeg не найден → Используется простое разбиение
├─ Ошибка ffmpeg → Возвращается исходный файл
└─ Ошибка I/O → Логируется, отправка ERROR ответа

Уровень 5: Отправка ответа
├─ GPIO17 не отзывается → Логируется, повтор
├─ Ошибка UART передачи → Логируется, повтор
└─ Serial.SerialException → Переподключение к порту
```

## 🔄 Восстановление после сбоев

```
Сценарий 1: Отключение питания
└─ При включении: systemd сервис перезапускает систему → IDLE

Сценарий 2: Обрыв RS-485 кабеля
└─ Система продолжает слушать (timeout обработан) → IDLE

Сценарий 3: Переполнение памяти
└─ Python выбросит MemoryError → логируется → graceful shutdown

Сценарий 4: Зависание rpicam-vid
└─ asyncio timeout срабатывает → процесс убивается → ERROR ответ

Сценарий 5: Повреждение видеофайла
└─ ffmpeg обработает как есть → warning в логи → отправка ответа
```
