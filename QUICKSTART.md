```
Raspberry Pi Zero 2W
```

## GPIO17 состояния

- **LOW (0V)** - режим приема (слушаем RS-485)
- **HIGH (3.3V)** - режим передачи (отправляем по RS-485)

## RS-485 схема подключения

```
Raspberry Pi Zero 2W     SN65HVD1781AQDRQ1     Линия RS-485
├─ GPIO17 ─────────────→ DE/RE
├─ GPIO14 (TX) ────────→ DI (Driver Input)
├─ GPIO15 (RX) ←────── RO (Receiver Output)
├─ GND ────────────────→ GND
│
└─ 3.3V ───┐
           ├─ Питание преобразователя (проверьте напряжение)
           │

│  B ───────► Линия B          │ (до 32 приборов)
```

## Частые вопросы

**Q: Почему 115200 бод?**  
A: Это стандартная и надежная скорость для отладки RS-485, которую обычно поддерживают USB-UART/USB-RS485 адаптеры.

**Q: Что если видео больше 50 МБ?**  
A: Система автоматически разбивает его на части с помощью ffmpeg.

**Q: Как изменить размер части?**  
A: Редактируйте в main.py: `CHUNK_SIZE_MB = 50`

**Q: Система работает в фоне?**  
A: Да, используйте systemd сервис (см. CONFIG.md)

**Q: Как отправить команду с компьютера?**  
A: Используйте `host_simulator.py` с USB RS-485 адаптером

## Поддерживаемые команды (для расширения)

Текущая версия поддерживает только payload `00 01` с CRC-32/ISO-HDLC в конце кадра.

Для добавления новых команд, отредактируйте в main.py:
```python
async def handle_start_command(self) -> None:
    # ... текущая логика

async def handle_stop_command(self) -> None:
    # ... ваша логика для stop

# И в main_loop:
if command == b"stop":
    await self.handle_stop_command()
```
